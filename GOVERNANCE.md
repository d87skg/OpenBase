# OpenBase Governance

## OBEP Process

1. Proposal: Submit OBEP as GitHub Issue
2. Discussion: Community review (min 2 weeks)
3. Vote: Maintainers vote (majority required)
4. RFC: Accepted proposals become RFC documents
5. Implementation: RFC implemented in openbase_core
6. Release: Included in next release

## What Requires OBEP
- Breaking changes to Stable Core schemas
- New required fields
- Removal of event types
- Changes to hash or signature algorithms

## What Does NOT Require OBEP
- New event types
- New optional fields
- New transport support
- Documentation
- Bug fixes

## Extensions
Policy, Compliance, Risk, Approval, Audit maintained in separate repositories with independent governance.
