# Terminology

## Purpose

Provides the single source of truth for all terms used in OpenBase protocol validation.

## Terms

### Requirement

A formal condition that an implementation must satisfy. Requirements are defined in protocol specifications and referenced by  + "REQ-XXX" +  identifiers.

### Specification

A document that defines protocol semantics. Examples: Evidence Specification, Execution Semantics.

### Implementation

A runtime that realizes the protocol. Example: OpenClaw Runtime.

### Runtime

The execution environment for an Agent. Runtimes produce Evidence and may be validated for Conformance.

### Evidence

A standardized, verifiable record of execution facts. Evidence is defined by the Evidence Specification.

### Verification

The process of independently validating Evidence against the protocol.

### Validation

The process of checking whether an Implementation conforms to the protocol.

### Replay

The reconstruction of an Execution from its Evidence stream.

### Conformance

The state of an Implementation that satisfies all mandatory requirements of the protocol.

### Conformance Report

A formal document that declares the Conformance status of an Implementation.

### Compatibility

The ability of different Implementations to interoperate under the same protocol.

### Certification

Formal recognition, issued by a Certification Authority, that an Implementation is Conformant.

### Gap

A documented discrepancy between the Specification and the Implementation.

### Traceability

The ability to trace a Requirement from its definition through Specification, Implementation, Test, Evidence, and Certification.

### Coverage

The degree to which Requirements are implemented, tested, and evidenced.
